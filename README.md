# EmbodiedIQ: 具身智能赛道投资信号雷达

[![Streamlit App](https://static.streamlit.io/badges/streamlit_badge_black_white.svg)](#)

> 一个面向具身智能（Embodied AI / 人形机器人）赛道的投资决策支持仪表盘，
> 整合融资、学术、专利、开源、人才等多维信号，帮助技术投资人快速扫描赛道动态。

## 🔍 为什么做这个项目？

在我于深圳天使投资引导基金主导具身智能赛道扫描期间，我完成了4份研究报告、
参与了ZBL等项目的投资尽调。但有一个痛点始终存在：**80%的研究时间浪费在数据收集上**——
融资动态、顶会论文、专利更新、开源项目进展散落在IT桔子、arXiv、Google Patents、GitHub等不同平台，
手工整合耗时巨大，且难以持续追踪。

EmbodiedIQ是我用数据科学方法解决这个痛点的尝试：它将多源异构信号整合到一个仪表盘，
让投资人把精力集中在判断而非收集上。

## 📊 核心功能

### 1. 赛道总览：三轴时序看板

融合融资趋势、论文发表趋势、专利申请趋势，判断赛道处于"期望膨胀期"还是"稳步复苏期"。

### 2. 公司活跃度排行

基于融资+学术+专利+开源+招聘的五维加权评分，识别最值得关注的早期项目。

### 3. 技术路线竞争格局

基于BERTopic对论文/专利的主题聚类，自动识别VLA模型、灵巧手、全身控制、仿真平台等技术方向的热度分布。

### 4. 竞品动态对比

选择2-3家目标公司，对比其融资节奏、专利布局、开源活跃度、招聘趋势。

## 🏗️ 技术架构

```
数据源（arXiv API + GitHub API + 手工维护）
    ↓
数据融合（Pandas + 实体对齐）
    ↓
主题建模（BERTopic）→ 技术路线自动分类
    ↓
信号评分（加权活跃度指数 + 去噪逻辑）
    ↓
可视化（Streamlit + Plotly交互仪表盘）
```

## 📈 数据覆盖

- **时间范围**：2023年1月 - 2026年4月
- **公司数量**：35+ 具身智能赛道核心公司
- **论文覆盖**：arXiv cs.RO/cs.AI/cs.CV 相关论文 400+
- **开源项目**：关键GitHub仓库 15+

## 🎯 使用场景

**场景1：赛道扫描**

> "具身智能赛道最近热度很高，但实际技术进展如何？是泡沫还是实质突破？"
> → 打开总览页，看融资趋势与论文/专利趋势是否同步增长。

**场景2：项目发现**

> "有哪些早期项目虽然还没拿到大额融资，但技术信号很强？"
> → 打开公司排行，筛选"论文/专利信号强但融资额低"的项目。

**场景3：竞品对标**

> "ZBL的技术路线和专利布局，与主要竞品相比处于什么位置？"
> → 打开竞品对比，选择ZBL和2-3家对标公司。

## ⚠️ 局限性与下一步

当前版本是MVP，主要限制：

1. **数据覆盖**：部分数据源依赖手工维护，覆盖面和实时性有限
2. **评分模型**：当前使用规则加权评分，尚未引入机器学习预测模型
3. **技术深度**：主题模型基于论文摘要，对技术细节的理解有限

下一步计划（希望在港中深DS硕士中深化）：

- 引入LLM进行专利/论文的深度技术标签提取
- 构建基于历史数据的融资预测模型
- 增加投后监控模块（被投企业异常信号检测）

## 🚀 快速开始

```bash
cd embodiediq

pip install -r requirements.txt
streamlit run app/app.py
```
